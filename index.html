1. Introduction to Python

Python is a high-level, interpreted programming language renowned for its readability, simplicity, and versatility. Created by Guido van Rossum and first released in 1991, Python's design philosophy emphasizes code readability with its notable use of significant indentation.

Why Learn Python?

  • Easy to Learn & Read: Python's syntax is clean and intuitive, resembling plain English, making it an excellent language for beginners.
  • Versatile & Powerful: Used in a wide array of applications, including web development, data science, machine learning, artificial intelligence, scientific computing, automation, and more.
  • Large & Active Community: A vast global community contributes to a rich ecosystem of libraries and frameworks, offering solutions for almost any problem.
  • Extensive Standard Library: Python comes with a "batteries included" standard library that provides tools for many common tasks.
  • Cross-Platform: Python code runs искусство (platform-independently) on Windows, macOS, Linux, and other operating systems.
  • High Demand: Python developers are highly sought after in the job market across various industries.

What can you do with Python?

  • Web Development: Frameworks like Django and Flask.
  • Data Analysis & Visualization: Libraries like Pandas, NumPy, Matplotlib, Seaborn.
  • Machine Learning & AI: Libraries like Scikit-learn, TensorFlow, PyTorch.
  • Automation & Scripting: Automate repetitive tasks on your computer.
  • Game Development: Libraries like Pygame.
  • Desktop GUIs: Libraries like Tkinter, PyQt, Kivy.
  • And much more!

Python Versions

There are two major Python versions: Python 2 and Python 3. Python 2 is legacy and no longer supported (official support ended in 2020). This tutorial focuses on Python 3, which is the present and future of Python. Ensure you install a recent version of Python 3 (e.g., 3.8 or newer).

Your First Python Program: "Hello, World!"

The traditional first program in any language is one that prints "Hello, World!" to the screen. In Python, it's remarkably simple:

print("Hello, World!")

To run this:

  1. Ensure Python 3 is installed.
  2. Open a text editor, type the code, and save it as a file (e.g., `hello.py`).
  3. Open a terminal or command prompt, navigate to the directory where you saved the file, and run: 
    python hello.py
  4. Alternatively, you can open the Python interpreter (type `python` or `python3` in your terminal) and type the command directly.

Setting up Your Environment

To start coding in Python, you'll need:

  1. Install Python: Download the latest version of Python 3 from python.org.
  2. Choose a Code Editor or IDE (Integrated Development Environment):
    • VS Code (Visual Studio Code): Free, powerful, highly extensible, and very popular. (Recommended)
    • PyCharm: Feature-rich IDE specifically for Python. (Community Edition is free)

2. Variables and Data Types

Variables are fundamental to programming. They act as named containers or labels for storing data values in memory. Python is dynamically typed, meaning you don't need to explicitly declare a variable's type; Python infers it from the value assigned.

Creating Variables

You create a variable by assigning a value to it using the assignment operator (`=`):

# Variable_name = value
age = 25
name = "Alice"
height_meters = 1.75
is_learning = True

Variable Naming Rules and Conventions:

  • Must start with a letter (a-z, A-Z) or an underscore (`_`).
  • Can be followed by letters, numbers (0-9), or underscores.
  • Are case-sensitive (e.g., `myVar` and `myvar` are different).
  • Cannot be a Python keyword (e.g., `if`, `for`, `class`).
  • Convention (PEP 8): Use `snake_case` for variable names (all lowercase with underscores separating words), e.g., `user_age`, `total_amount`.

Multiple Assignment:

Python allows you to assign values to multiple variables in a single line:

x, y, z = 10, "Python", 3.14
print(x)         # Output: 10
print(y)         # Output: Python

a = b = c = "Same Value"
print(b)         # Output: Same Value

Core Data Types

Python has several built-in data types. Here are the most common ones:

1. Numeric Types:

  • Integer (int): Whole numbers, positive or negative, without decimals. E.g., 10, -3, 0, 1000000.
  • Float (float): Numbers with a decimal point or in exponential (E) notation. E.g., 3.14, -0.5, 2.5e2 (which is 2.5 * 102 = 250.0).
  • Complex (complex): Numbers with a real and an imaginary part, written as `x + yj`. E.g., `3+5j`, `2-7j`. (Less common in basic tutorials).
count = 100       # int
price = 19.99     # float
temperature = -5.0  # float

2. Sequence Types:

  • String (str): An ordered sequence of characters, used for text. Enclosed in single ('), double ("), or triple (''' or """ for multi-line) quotes. Strings are immutable (cannot be changed after creation). 
    message = "Hello, Python!"
char = 'A'
multiline_str = '''This is a
multi-line string.'''
  • List (list): An ordered, mutable (changeable) collection of items. Items can be of different types. Defined using square brackets `[]`. (More detail in Data Structures section)
  • Tuple (tuple): An ordered, immutable (unchangeable) collection of items. Defined using parentheses `()`. (More detail in Data Structures section)

3. Mapping Type:

  • Dictionary (dict): An unordered (in Python < 3.7, ordered in Python 3.7+) collection of key-value pairs. Mutable. Defined using curly braces `{}`. (More detail in Data Structures section)

4. Set Types:

  • Set (set): An unordered collection of unique items. Mutable. Defined using curly braces `{}` or `set()`. (More detail in Data Structures section)
  • Frozenset (frozenset): An immutable version of a set.

5. Boolean Type:

  • Boolean (bool): Represents one of two truth values: True or False. Often the result of comparisons.
is_active = True
has_permission = False

6. None Type:

  • NoneType (None): Represents the absence of a value or a null value. A variable that has not been assigned a value yet (or explicitly set to `None`) is of `NoneType`.
user_email = None
if user_email is None:
    print("Email not provided.")

Type Checking and Conversion

You can check the type of a variable using the built-in type() function, and convert between compatible types using functions like int(), float(), str(), list(), etc.

quantity = "10"  # This is a string
print(type(quantity)) # Output: <class 'str'>

# Convert string to integer
num_quantity = int(quantity)
print(type(num_quantity)) # Output: <class 'int'>
print(num_quantity * 5) # Output: 50

age = 30
age_message = "I am " + str(age) + " years old." # Convert int to str for concatenation
print(age_message) # Output: I am 30 years old.
Dynamic Typing: Python's dynamic typing is convenient but means you need to be mindful of the types of data your variables hold, especially when performing operations, to avoid `TypeError` exceptions.

3. Operators in Python

Operators are special symbols or keywords in Python that perform operations on values and variables (operands). Understanding operators is crucial for performing calculations, making decisions, and manipulating data.

1. Arithmetic Operators

Used to perform mathematical calculations.

a = 10
b = 3

print(f"Addition (a + b):         {a + b}")  # Output: 13
print(f"Subtraction (a - b):      {a - b}")  # Output: 7
print(f"Multiplication (a * b):   {a * b}")  # Output: 30
print(f"Division (a / b):         {a / b}")  # Output: 3.333... (float division)
print(f"Floor Division (a // b):  {a // b}") # Output: 3 (integer division, discards remainder)
print(f"Modulus (a % b):          {a % b}")  # Output: 1 (remainder of division)
print(f"Exponentiation (a ** b):  {a ** b}") # Output: 1000 (a to the power of b)

2. Comparison (Relational) Operators

Used to compare two values. They return a Boolean value (True or False).

x = 5
y = 10

print(f"Equal (x == y):           {x == y}") # Output: False
print(f"Not Equal (x != y):       {x != y}") # Output: True
print(f"Greater Than (x > y):     {x > y}") # Output: False
print(f"Less Than (x < y):        {x < y}") # Output: True
print(f"Greater or Equal (x >= y):{x >= y}") # Output: False
print(f"Less or Equal (x <= y):   {x <= y}") # Output: True (Original comment was: Actually False for 5 <= 10, typo in comment! Corrected: 5 <= 10 is True)
print(f"Less or Equal (5 <= 10):  {5 <= 10}") # Output: True (Correct example)

3. Logical Operators

Used to combine conditional statements. They also return True or False.

is_sunny = True
is_warm = False

# and: True if both operands are true
print(f"is_sunny and is_warm: {is_sunny and is_warm}") # Output: False

# or: True if at least one operand is true
print(f"is_sunny or is_warm:  {is_sunny or is_warm}")  # Output: True

# not: True if operand is false (inverts the boolean value)
print(f"not is_warm:          {not is_warm}")          # Output: True
Short-circuiting: Logical operators and and or use short-circuit evaluation. If the result can be determined from the first operand, the second is not evaluated (e.g., if `A` is `False` in `A and B`, `B` is not checked).

4. Assignment Operators

Used to assign values to variables. We've already seen `=`, but there are shorthand versions.

counter = 0
counter += 1      # Equivalent to: counter = counter + 1;  counter is now 1
print(f"counter: {counter}")

total = 100
total -= 10     # Equivalent to: total = total - 10;   total is now 90
print(f"total: {total}")

product = 5
product *= 3    # Equivalent to: product = product * 3; product is now 15
print(f"product: {product}")

# Others include: /=, //=, %=, **=

5. Membership Operators

Used to test if a sequence (like a string, list, or tuple) contains a specific value. Return True or False.

my_list = [1, 2, 3, "apple"]
message = "Hello World"

# in: True if value is found in the sequence
print(f"3 in my_list:          {3 in my_list}")      # Output: True
print(f"'World' in message:    {'World' in message}")  # Output: True

# not in: True if value is NOT found in the sequence
print(f"4 not in my_list:      {4 not in my_list}")  # Output: True
print(f"'Python' not in message: {'Python' not in message}")# Output: True

6. Identity Operators

Used to compare the memory locations of two objects (i.e., if they are the same object, not just if they are equal in value). Return True or False.

list_a = [1, 2]
list_b = [1, 2]
list_c = list_a

# is: True if both variables point to the same object in memory
print(f"list_a is list_b: {list_a is list_b}") # Output: False (different objects, though values are same)
print(f"list_a is list_c: {list_a is list_c}") # Output: True (list_c is an alias for list_a)

# is not: True if both variables point to different objects
print(f"list_a is not list_b: {list_a is not list_b}") # Output: True
is vs ==: Use == to check if two variables have the same value. Use is to check if they refer to the exact same object instance in memory. For immutable types like small integers and strings, Python might reuse objects, so is can sometimes be True even for independently created variables, but don't rely on this behavior for correctness; use == for value comparison.

Operator Precedence

Python evaluates expressions based on operator precedence (similar to mathematical order of operations). Parentheses () can be used to override the default order.

Example: ** (exponentiation) has higher precedence than * (multiplication), which is higher than + (addition).

result = 2 + 3 * 4       # 3 * 4 is 12, then 2 + 12 = 14
print(f"Result: {result}")  # Output: 14

result_paren = (2 + 3) * 4 # 2 + 3 is 5, then 5 * 4 = 20
print(f"Result with parentheses: {result_paren}") # Output: 20

4. Control Flow

Control flow statements determine the order in which your program's statements are executed. They allow you to make decisions and repeat actions, making your programs dynamic and intelligent.

Conditional Statements (if, elif, else)

These statements execute different blocks of code based on whether a condition is true or false.

age = 19

if age >= 18:
    print("You are an adult.")
    if age >= 21:
        print("You can also buy alcohol in some regions.") # Nested if
elif age >= 13:
    print("You are a teenager.")
else:
    print("You are a child.")

Ternary Operator (Conditional Expression)

A concise way to write a simple if-else statement in one line.

# value_if_true if condition else value_if_false
score = 75
result = "Pass" if score >= 50 else "Fail"
print(result)  # Output: Pass

Loops

Loops are used to execute a block of code repeatedly.

1. for Loop

Iterates over a sequence (like a list, tuple, dictionary, set, or string) or other iterable objects.

Iterating over a list:
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print(fruit)
Iterating over a string:
for char in "Python":
    print(char)
The range() function:

Often used with for loops to iterate a specific number of times.

# range(stop) - 0 up to (but not including) stop
for i in range(5):  # 0, 1, 2, 3, 4
    print(i)

# range(start, stop) - start up to (but not including) stop
for i in range(2, 6): # 2, 3, 4, 5
    print(i)

# range(start, stop, step) - start up to stop, incrementing by step
for i in range(0, 10, 2): # 0, 2, 4, 6, 8
    print(i)
The enumerate() function:

Provides both the index and the value during iteration.

colors = ["red", "green", "blue"]
for index, color in enumerate(colors):
    print(f"Index {index}: {color}")

2. while Loop

Repeats a block of code as long as a specified condition is true.

count = 0
while count < 5:
    print(f"Count is: {count}")
    count += 1 # Important: update the condition variable to avoid infinite loop
print("Loop finished.")
Infinite Loops: Be careful with while loops. If the condition never becomes false, the loop will run forever. You can usually stop an infinite loop by pressing `Ctrl+C` in the terminal.

Loop Control Statements

These statements change the execution of a loop from its normal sequence.

  • break: Terminates the current loop and transfers control to the statement immediately following the loop.
  • continue: Skips the rest of the code inside the loop for the current iteration and proceeds to the next iteration.
  • pass: Acts as a placeholder. It does nothing when executed. Useful when a statement is syntactically required but you don't want any code to execute.
print("\n--- break example ---")
for i in range(10):
    if i == 5:
        break  # Exit loop when i is 5
    print(i) # Prints 0, 1, 2, 3, 4

print("\n--- continue example ---")
for i in range(5):
    if i == 2:
        continue # Skip iteration when i is 2
    print(i) # Prints 0, 1, 3, 4

print("\n--- pass example ---")
for i in range(3):
    if i == 1:
        pass # Placeholder, does nothing
    print(i) # Prints 0, 1, 2

else Clause in Loops

Python loops (both for and while) can have an optional else clause. The code in the else block is executed if the loop completes normally (i.e., not terminated by a break statement).

print("\n--- for...else example ---")
for num in [1, 3, 5]:
    print(f"Checking {num}")
    if num % 2 == 0:
        print("Even number found!")
        break
else: # Executed if loop completes without break
    print("No even numbers found.")

print("\n--- while...else example ---")
attempts = 0
while attempts < 3:
    print(f"Attempt {attempts + 1}")
    # password = input("Enter password: ")
    # if password == "secret":
    #     print("Access granted")
    #     break
    attempts += 1
else: # Executed if loop completes without break
    print("Too many failed attempts (simulated).")

5. Python Data Structures

Data structures are ways of organizing and storing data so that it can be accessed and modified efficiently. Python provides several powerful built-in data structures.

1. Lists

A list is an ordered, mutable (changeable) collection of items. Lists can contain items of different data types.

Creating Lists:

empty_list = []
numbers = [1, 2, 3, 4, 5]
mixed_list = [1, "hello", 3.14, True]
list_of_lists = [[1, 2], [3, 4]]

Accessing Elements (Indexing and Slicing):

my_list = ['a', 'b', 'c', 'd', 'e']
print(f"First element: {my_list[0]}")         # Output: 'a' (0-based indexing)
print(f"Last element: {my_list[-1]}")        # Output: 'e' (negative indexing)
print(f"Slice (1 to 3): {my_list[1:4]}")   # Output: ['b', 'c', 'd'] (index 4 is exclusive)
print(f"Slice from start: {my_list[:3]}")   # Output: ['a', 'b', 'c']
print(f"Slice to end: {my_list[2:]}")       # Output: ['c', 'd', 'e']

Modifying Lists:

my_list = [10, 20, 30]
my_list[1] = 25                  # Change element at index 1
print(f"Modified list: {my_list}") # Output: [10, 25, 30]

my_list.append(40)             # Add to the end
print(f"After append: {my_list}")  # Output: [10, 25, 30, 40]

my_list.insert(1, 15)           # Insert 15 at index 1
print(f"After insert: {my_list}")  # Output: [10, 15, 25, 30, 40]

removed_item = my_list.pop()    # Remove and return last item (40)
print(f"After pop: {my_list}, Removed: {removed_item}")

my_list.remove(15)              # Remove first occurrence of 15
print(f"After remove: {my_list}")

del my_list[0]                # Delete item at index 0
print(f"After del: {my_list}")

Other List Methods:

extend(iterable), sort(), reverse(), copy(), clear(), count(item), index(item).

List Comprehensions:

A concise way to create lists.

squares = [x**2 for x in range(1, 6)] # [1, 4, 9, 16, 25]
print(f"Squares: {squares}")

even_numbers = [x for x in range(10) if x % 2 == 0] # [0, 2, 4, 6, 8]
print(f"Even numbers: {even_numbers}")

2. Tuples

A tuple is an ordered, immutable (unchangeable) collection of items. Once a tuple is created, you cannot change its content.

Creating Tuples:

empty_tuple = ()
single_item_tuple = ("hello",) # Note the trailing comma!
my_tuple = (1, "apple", 3.14)
# Parentheses are optional in some cases for tuple creation:
another_tuple = 10, 20, "_world_"

Accessing Elements:

Same as lists (indexing and slicing).

print(f"First element of my_tuple: {my_tuple[0]}") # Output: 1

Why use Tuples?

  • For data that should not change (immutability ensures data integrity).
  • Can be used as keys in dictionaries (because they are immutable).
  • Often used for fixed collections of items, like coordinates (x, y).
  • Slightly more memory efficient and faster to iterate over than lists (for very large collections).

Tuple Packing and Unpacking:

# Packing
point = (10, 20)

# Unpacking
x, y = point
print(f"x: {x}, y: {y}") # Output: x: 10, y: 20

3. Dictionaries

A dictionary is an unordered (Python < 3.7) or ordered (Python 3.7+) collection of key-value pairs. Keys must be unique and immutable (e.g., strings, numbers, tuples). Dictionaries are mutable.

Creating Dictionaries:

empty_dict = {}
student = {
    "name": "Alice",
    "age": 21,
    "major": "Computer Science"
}
# Using dict() constructor
car = dict(brand="Ford", model="Mustang", year=1964)

Accessing and Modifying Values:

print(f"Student name: {student['name']}") # Access using key
print(f"Student major (get): {student.get('major', 'N/A')}") # Safe access with default

student['age'] = 22                 # Modify existing value
student['grade'] = 'A'                # Add new key-value pair
print(f"Updated student: {student}")

removed_major = student.pop('major') # Remove item by key and return its value
print(f"After pop: {student}, Removed: {removed_major}")

del student['grade']            # Delete item by key
print(f"After del: {student}")

Dictionary Methods:

keys(), values(), items() (for iterating), update(other_dict), clear(). 
print(f"Keys: {student.keys()}")
print(f"Values: {student.values()}")
print(f"Items: {student.items()}") # Returns view objects of (key, value) tuples

for key, value in student.items():
    print(f"{key}: {value}")

Dictionary Comprehensions:

squared_numbers = {x: x**2 for x in range(1, 6)}
# Output: {1: 1, 2: 4, 3: 9, 4: 16, 5: 25}
print(f"Squared numbers dict: {squared_numbers}")

4. Sets

A set is an unordered collection of unique, immutable items. Sets are mutable (you can add/remove items). They are useful for membership testing, removing duplicates, and mathematical set operations.

Creating Sets:

empty_set = set() # Important: {} creates an empty dictionary, not a set
my_set = {1, 2, 3, "apple", 2} # Duplicates are automatically removed
print(f"My set: {my_set}") # Output might be {1, 2, 3, 'apple'} (order not guaranteed)

list_with_duplicates = [1, 2, 2, 3, 4, 4, 4]
unique_items_set = set(list_with_duplicates)
print(f"Set from list: {unique_items_set}") # Output: {1, 2, 3, 4}

Set Operations:

set_a = {1, 2, 3, 4}
set_b = {3, 4, 5, 6}

// Union: items in either set_a or set_b or both
print(f"Union (A | B): {set_a | set_b}") # or set_a.union(set_b)

// Intersection: items common to both set_a and set_b
print(f"Intersection (A & B): {set_a & set_b}") # or set_a.intersection(set_b)

// Difference: items in set_a but not in set_b
print(f"Difference (A - B): {set_a - set_b}") # or set_a.difference(set_b)
print(f"Difference (B - A): {set_b - set_a}") # or set_b.difference(set_a)

// Symmetric Difference: items in either A or B, but not both
print(f"Symmetric Diff (A ^ B): {set_a ^ set_b}") # or set_a.symmetric_difference(set_b)

Set Methods:

add(item), remove(item) (raises KeyError if not found), discard(item) (no error if not found), pop() (removes and returns an arbitrary item), clear().

6. String Manipulation

Strings are sequences of characters and are one of the most commonly used data types. Python provides a rich set of tools for working with strings. Remember, strings are immutable.

Creating Strings

single_quotes = 'Hello'
double_quotes = "World"
triple_quotes_multiline = '''This is
a multi-line
string.'''
empty_string = ""

String Concatenation and Repetition

first_name = "John"
last_name = "Doe"
full_name = first_name + " " + last_name # Concatenation
print(f"Full Name: {full_name}") # Output: John Doe

separator = "-" * 10 # Repetition
print(f"Separator: {separator}") # Output: ----------

Accessing Characters (Indexing and Slicing)

Works just like lists and tuples.

text = "Python"
print(f"First char: {text[0]}")   # Output: P
print(f"Last char: {text[-1]}")    # Output: n
print(f"Slice: {text[1:4]}")    # Output: yth (from index 1 up to, not including, 4)
print(f"Slice with step: {text[0:6:2]}") # Output: Pto (every 2nd char)
print(f"Reverse string: {text[::-1]}") # Output: nohtyP

Common String Methods

String methods return new strings (due to immutability).

s = "  Hello, World!  "

print(f"Length: {len(s)}") # Output: 19 (general function, not a method)
print(f"Lowercase: '{s.lower()}'") # Output: '  hello, world!  '
print(f"Uppercase: '{s.upper()}'") # Output: '  HELLO, WORLD!  '
print(f"Strip whitespace: '{s.strip()}'") # Output: 'Hello, World!' (removes leading/trailing)
print(f"Lstrip: '{s.lstrip()}'") # Output: 'Hello, World!  ' (removes leading)
print(f"Rstrip: '{s.rstrip()}'") # Output: '  Hello, World!' (removes trailing)

print(f"Replace 'World' with 'Python': '{s.replace('World', 'Python')}'")
# Output: '  Hello, Python!  '

csv_data = "apple,banana,cherry"
fruits_list = csv_data.split(',')
print(f"Split string: {fruits_list}") # Output: ['apple', 'banana', 'cherry']

words = ['Hello', 'Python', 'World']
sentence = " ".join(words)
print(f"Joined list: '{sentence}'") # Output: 'Hello Python World'

print(f"Starts with '  H': {s.startswith('  H')}") # Output: True
print(f"Ends with '!  ': {s.endswith('!  ')}") # Output: True

print(f"Find 'o': {s.find('o')}") # Output: 6 (index of first occurrence, -1 if not found)
print(f"Count 'l': {s.count('l')}") # Output: 3

Other useful methods: capitalize(), title(), isalpha(), isdigit(), isalnum(), isspace(), etc.

String Formatting

Creating strings by embedding expressions or variables inside them.

1. f-Strings (Formatted String Literals) - Recommended

Available in Python 3.6+. Concise and readable.

name = "Alice"
age = 30
greeting = f"Hello, my name is {name} and I am {age} years old."
print(greeting)
# Output: Hello, my name is Alice and I am 30 years old.

pi = 3.14159265
print(f"Pi to 2 decimal places: {pi:.2f}") # Formatting within f-string
# Output: Pi to 2 decimal places: 3.14

2. str.format() Method

Available in Python 2.6+ and 3.x. More verbose than f-strings.

item = "book"
price = 19.99
message1 = "The {} costs ${}.".format(item, price)
print(message1)

message2 = "The {product} costs ${cost:.2f}.".format(product=item, cost=price)
print(message2)

3. % Operator (Older style)

Similar to C's `printf`. Generally not recommended for new code.

name = "Bob"
value = 42
old_style = "Name: %s, Value: %d" % (name, value)
print(old_style)

7. Functions

Functions are blocks of reusable code designed to perform a specific task. They help make your code more organized, modular, readable, and reduce repetition (DRY - Don't Repeat Yourself).

Defining a Function

Use the def keyword, followed by the function name, parentheses () for parameters, and a colon :. The code block within the function must be indented.

def greet(name):
    """This is a docstring. It explains what the function does."""
    print(f"Hello, {name}!")

Docstrings:

The first statement in a function body can optionally be a string literal, which is the function's documentation string or "docstring". It's good practice to include docstrings to explain what your function does, its parameters, and what it returns.

Calling a Function

To execute a function, you "call" it by its name followed by parentheses, passing any required arguments.

greet("Alice")  # Output: Hello, Alice!
greet("Bob")    # Output: Hello, Bob!

Parameters and Arguments

  • Parameter: A variable listed inside the parentheses in the function definition.
  • Argument: The actual value that is sent to the function when it is called.

Return Values

Functions can return a value to the caller using the return statement. If no return statement is specified, or if it's used without an expression, the function implicitly returns None.

def add_numbers(x, y):
    return x + y

sum_result = add_numbers(5, 3)
print(f"Sum: {sum_result}")  # Output: Sum: 8

def no_return_value():
    print("This function doesn't explicitly return.")

result = no_return_value()
print(f"Result of no_return_value: {result}") # Output: Result of no_return_value: None

Returning Multiple Values:

A function can return multiple values by returning them as a tuple (or list, or other collection).

def get_coordinates():
    return 10, 20 # Returns a tuple (10, 20)

x_coord, y_coord = get_coordinates() # Tuple unpacking
print(f"X: {x_coord}, Y: {y_coord}") # Output: X: 10, Y: 20

Types of Arguments

1. Positional Arguments:

Passed to a function in the order they are defined.

def describe_pet(animal_type, pet_name):
    print(f"I have a {animal_type} named {pet_name}.")

describe_pet("dog", "Buddy")

2. Keyword Arguments:

Passed using `name=value` syntax. Order doesn't matter if keywords are used.

describe_pet(pet_name="Lucy", animal_type="cat")

3. Default Parameter Values:

You can specify default values for parameters. Parameters with defaults must come after those without defaults.

def power(base, exponent=2): # exponent has a default value
    return base ** exponent

print(f"3 squared: {power(3)}")          # Output: 9 (uses default exponent)
print(f"2 cubed: {power(2, 3)}")      # Output: 8 (overrides default)
Mutable Default Arguments: Be careful when using mutable objects (like lists or dictionaries) as default arguments. The default is created only once when the function is defined. If modified, subsequent calls will use the modified default. It's often better to default to `None` and create the mutable object inside the function. 
# Potentially problematic:
def add_to_list_bad(item, my_list=[]):
    my_list.append(item)
    return my_list

# Better approach:
def add_to_list_good(item, my_list=None):
    if my_list is None:
        my_list = []
    my_list.append(item)
    return my_list

4. Variable-Length Arguments (*args and **kwargs)

*args (Arbitrary Positional Arguments):

Allows a function to accept any number of positional arguments. These are collected into a tuple.

def print_all(*args):
    print(f"Arguments received (tuple): {args}")
    for arg in args:
        print(arg)

print_all(1, "hello", 3.14)
**kwargs (Arbitrary Keyword Arguments):

Allows a function to accept any number of keyword arguments. These are collected into a dictionary.

def print_info(**kwargs):
    print(f"Keyword arguments (dict): {kwargs}")
    for key, value in kwargs.items():
        print(f"{key}: {value}")

print_info(name="Alice", age=25, city="New York")

You can use regular parameters, `*args`, and `**kwargs` together in a function definition, in that order.

Scope of Variables (LEGB Rule)

Scope determines the visibility and lifetime of a variable.

  • Local Scope: Variables defined inside a function are local to that function. They cannot be accessed outside it.
  • Global Scope: Variables defined outside of any function are global. They can be accessed (but not directly modified without `global` keyword) from within functions.
global_var = "I am global"

def my_function():
    local_var = "I am local"
    print(local_var)
    print(global_var) # Can access global_var

my_function()
# print(local_var) # This would cause a NameError

def modify_global():
    global global_var # Declare intent to modify global variable
    global_var = "Modified global"

modify_global()
print(global_var) # Output: Modified global

The LEGB rule describes the order Python searches for names: Local, Enclosing function locals, Global, Built-in.

Lambda Functions (Anonymous Functions)

Small, single-expression anonymous functions defined using the lambda keyword. Often used for short operations where a full function definition is too verbose.

# lambda arguments: expression
add = lambda a, b: a + b
print(f"Lambda add(5, 3): {add(5, 3)}") # Output: 8

points = [(1, 2), (3, 1), (5, 4)]
# Sort points by the second element of each tuple
points.sort(key=lambda point: point[1])
print(f"Sorted points: {points}") # Output: [(3, 1), (1, 2), (5, 4)]

8. Error and Exception Handling

Errors are inevitable in programming. Python uses exceptions to manage errors that occur during program execution. Proper exception handling makes your code more robust and user-friendly.

Syntax Errors vs. Exceptions

  • Syntax Errors (Parsing Errors): Occur when your code violates Python's syntax rules. The interpreter catches these before execution begins. 
    # Example of a syntax error (missing colon)
# if x > 5
#    print("x is greater than 5")
  • Exceptions (Runtime Errors): Occur during the execution of the program if something unexpected happens, even if the syntax is correct. 
    # Example of a runtime error (ZeroDivisionError)
# result = 10 / 0

# Example of a runtime error (TypeError)
# result = "hello" + 5

Common Built-in Exceptions

  • TypeError: Operation or function applied to an object of inappropriate type.
  • ValueError: Operation or function receives an argument with the right type but an inappropriate value.
  • NameError: A local or global name is not found.
  • IndexError: A sequence subscript is out of range.
  • KeyError: A dictionary key is not found.
  • FileNotFoundError: Attempting to open a file that does not exist.
  • ZeroDivisionError: Division or modulo by zero.
  • AttributeError: Attempting to access an attribute or method that an object doesn't have.
  • ImportError: The `import` statement fails to find the module definition.

Handling Exceptions: The try-except Block

You can handle exceptions using a try-except block. Code that might raise an exception is placed in the try block. If an exception occurs, the code in the corresponding except block is executed.

try:
    numerator = 10
    denominator = int(input("Enter a denominator: "))
    result = numerator / denominator
    print(f"Result: {result}")
except ZeroDivisionError:
    print("Error: Cannot divide by zero!")
except ValueError:
    print("Error: Please enter a valid integer for the denominator.")
except Exception as e: # Catch any other exceptions (general)
    print(f"An unexpected error occurred: {e}")
print("Program continues after try-except block.")

It's good practice to catch specific exceptions first, followed by more general ones. Catching `Exception` (or a bare `except:`) will catch almost all exceptions, which can sometimes hide bugs.

The else Clause

The optional else block is executed if the try block completes without raising any exceptions.

try:
    num = int(input("Enter a number: "))
except ValueError:
    print("That was not a valid number.")
else:
    print(f"You entered {num}. Square is {num**2}.")

The finally Clause

The optional finally block is always executed, whether an exception occurred or not. It's typically used for cleanup actions, like closing files or releasing resources.

file = None # Initialize file to None
try:
    file = open("my_data.txt", "r")
    content = file.read()
    print(content)
except FileNotFoundError:
    print("Error: my_data.txt not found.")
except Exception as e:
    print(f"An error occurred: {e}")
finally:
    if file: # Check if file was successfully opened
        file.close()
        print("File closed.")
    else:
        print("File was not opened, no need to close.")

Raising Exceptions (raise)

You can explicitly raise an exception using the raise keyword. This is useful for signaling errors in your own functions based on certain conditions.

def validate_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative.")
    elif age < 18:
        raise ValueError("User must be 18 or older.")
    print("Age is valid.")

try:
    validate_age(25)   # Valid
    validate_age(-5)    # This will raise ValueError
except ValueError as e:
    print(f"Validation Error: {e}")

You can raise built-in exceptions or define your own custom exceptions by creating a new class that inherits from `Exception` or one of its subclasses.

9. Modules and Packages (Basic Intro)

As your programs grow larger, it's beneficial to organize your code into manageable pieces. Python's modules and packages help achieve this by promoting code reusability and organization.

What is a Module?

A module is simply a Python file (with a .py extension) containing Python definitions and statements (functions, classes, variables). Modules allow you to logically organize your Python code. Grouping related code into a module makes the code easier to understand, use, and maintain.

For example, if you have a set of mathematical functions, you could put them in a file named `mymath.py`. This file `mymath.py` is a module.

Importing Modules

To use the definitions from a module in another Python script or in the interactive interpreter, you need to `import` it.

1. import module_name

This imports the entire module. To access its definitions, you need to use dot notation: `module_name.definition_name`.

# Assume we have a module named 'math_operations.py':
# def add(x, y): return x + y
# PI = 3.14159

import math_operations # If math_operations.py is in the same directory or Python path

result = math_operations.add(5, 3)
print(f"Sum: {result}")
print(f"PI: {math_operations.PI}")

2. from module_name import definition1, definition2

This imports specific definitions (functions, variables, classes) directly into the current namespace. You can then use them without the module prefix.

from math_operations import add, PI

result = add(10, 7) # No module_name prefix needed
print(f"Sum: {result}")
print(f"PI: {PI}")

3. from module_name import * (Wildcard Import)

This imports all names from the module into the current namespace. **Generally discouraged** because it can lead to name collisions and makes it unclear where a definition comes from, reducing code readability.

# from math_operations import * # Avoid if possible
# result = add(1, 2)

4. Importing with an Alias (as)

You can give an imported module or definition an alias (a different name) in your current script. This is useful for brevity or to avoid name conflicts.

import math_operations as mo
from math_operations import add as custom_add

sum1 = mo.add(2, 2)
sum2 = custom_add(3, 3)
print(f"Sum1: {sum1}, Sum2: {sum2}")

The Python Standard Library

Python comes with a vast standard library, which is a collection of modules providing a wide range of functionalities. You don't need to install these separately; they are part of your Python installation.

Example: math module

Provides mathematical functions.

import math

print(f"Square root of 16: {math.sqrt(16)}") # Output: 4.0
print(f"Value of pi: {math.pi}")
print(f"Ceiling of 2.3: {math.ceil(2.3)}")   # Output: 3
print(f"Floor of 2.7: {math.floor(2.7)}")    # Output: 2

Example: random module

Provides functions for generating random numbers.

import random

random_int = random.randint(1, 10) # Random integer between 1 and 10 (inclusive)
print(f"Random integer: {random_int}")

random_float = random.random() # Random float between 0.0 (inclusive) and 1.0 (exclusive)
print(f"Random float: {random_float}")

my_list = ['apple', 'banana', 'cherry']
random_choice = random.choice(my_list)
print(f"Random choice from list: {random_choice}")

Other useful standard library modules include datetime (for dates and times), os (for interacting with the operating system), sys (for system-specific parameters and functions), json (for working with JSON data), and many more.

What are Packages?

Packages are a way of structuring Python's module namespace by using "dotted module names". A package is essentially a directory containing other modules and an `__init__.py` file (which can be empty). The `__init__.py` file tells Python that the directory should be treated as a package.

For example, you could have a package structure like this:

mypackage/
    __init__.py
    module1.py
    module2.py
    subpackage/
        __init__.py
        submodule1.py

You could then import them like:

import mypackage.module1
from mypackage.subpackage import submodule1

Packages help organize large projects with many modules.

Third-Party Packages and pip

Beyond the standard library, there's a vast ecosystem of third-party packages created by the Python community, available on the Python Package Index (PyPI). You can install these packages using a tool called pip, which comes with most Python installations.

For example, to install a popular package like `requests` (for making HTTP requests):

# In your terminal or command prompt:
pip install requests

Once installed, you can import and use it in your Python scripts just like standard library modules.

import requests

response = requests.get('https://api.github.com')
print(f"GitHub API status: {response.status_code}")

This is just a brief introduction. Exploring modules and packages is a key part of becoming a proficient Python developer.
